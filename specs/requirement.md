## 一、棋盤設計

### 1.1 基本結構

- 16 格環形棋盤，以 5×5 grid 外圈排列
- 格子編號 1～16，順時針方向
- 棋子移動到第 16 格後，下一步回到第 1 格（環形循環）

## 三、遊戲回合流程

遊戲採「大輪制」：6 隊玩家依序行動完畢後，進入鬼王回合，稱為一大輪。

### 3.1 玩家回合流程

> 每隊輪到自己時，依以下順序執行：

**步驟 1 — 結算上一輪任務（若有）**

若本隊上一輪落在任務格並出發，本輪行動開始時先公告結果：

- **任務成功**：骰骰子（1～6），點數即為對鬼王造成的基礎傷害。若本隊擁有「雙倍標記」，傷害 ×2，標記消耗。
- **任務失敗**：無任何效果。

**步驟 2 — 擲骰移動**

擲骰子（1～6），棋子向前移動對應格數（環形，超過 16 溢出回到 1）。

**步驟 3 — 執行格子效果**

依落地格類型：

| 格子類型 | 效果 |
|:--------:|:-----|
| 任務格（★ / ×） | 大字公告任務內容，宣告「出發執行任務！」，**結果留待下一輪揭曉** |
| 抽卡格（◈） | 從 4 張面朝下的牌中選 1 張翻開，效果立即生效 |

**步驟 4 — 換下一位**

點擊「換下一位」，輪到下一隊；若 6 隊皆完成，進入鬼王回合。

> 系統會自動跳過陣亡（HP = 0）的隊伍。

---

### 3.2 鬼王回合流程

**步驟 1 — 檢查狀態**

若鬼王持有「白天來了」標記，本回合暫停行動，標記消耗，直接進入下一大輪。

**步驟 2 — 擲骰移動**

擲骰子（1～6），鬼王棋子向前移動對應格數（同樣環形循環）。

**步驟 3 — 攻擊**

系統自動計算所有**存活隊伍**與鬼王的環形最短距離，選出最近的 **2 支隊伍**作為攻擊目標。

> **距離算法**：`min(|位置A - 位置B|, 16 - |位置A - 位置B|)`
>
> **平手處理**：若多隊距離相同，以 Team ID 較小者優先（選滿 2 隊為止）。

再擲骰子（1～6），點數作為基礎傷害，乘以鬼王傷害倍率（預設 ×1）後對目標兩隊各自扣除 HP。

- 擁有「護盾標記」的隊伍可抵擋此次傷害，護盾消耗。
- 若鬼王持有「夜晚來臨」標記，傷害 ×2，標記消耗。

**步驟 4 — 大輪結束**

點擊「換下一位」，全局回合數 +1，回到 Team 1 開始新一大輪。

## 四、卡片系統

落在**抽卡格（◈）**時，系統有 50% 機率抽選【機會卡】、50% 機率抽選【命運卡】。

### 4.1 機會卡（Chance Cards）— 對玩家有利

| 卡名 | 效果說明 |
|:-----|:---------|
| 死者甦醒 | 復活一隊已陣亡（HP = 0）的小組，復活後 HP 重置為 **5**。若多隊陣亡，由工作人員指定復活哪一隊。若無陣亡隊伍則無效。 |
| 生命藥水 | 本組回復 **5 HP**（上限 10 HP）。 |
| 防護盾 | 本組獲得「護盾標記」，下次鬼王攻擊對本組**無效**（觸發後標記消耗）。 |
| 強化攻擊 | 本組獲得「雙倍標記」，任務成功時對鬼王的傷害 **×2**（觸發後消耗；若本輪未觸發則標記保留至下次）。 |

### 4.2 命運卡（Fate Cards）— 對鬼王有利

| 卡名 | 效果說明 |
|:-----|:---------|
| 夜晚來臨 | 鬼王獲得標記，**下一回合攻擊傷害 ×2**（攻擊後消耗）。 |
| 白天來了 | 鬼王獲得標記，**下一回合暫停行動一次**（輪到鬼王時消耗）。 |
| 鬼放大招 | 無慘立即對場上**所有存活隊伍**造成 **3 點傷害**（護盾可抵擋；傷害不計入各隊 `damageDealt`）。 |
| 真是個平安的一天 | 無事發生。 |

## 五、勝負條件

### 5.1 玩家勝利

鬼王 HP 歸零後，觸發【最終一擊】儀式畫面：

1. 全螢幕黑底，大字顯示：「鬼王無慘血量歸零！」
2. 引導全場齊喊：**「鬼殺隊──斬！」**
3. 工作人員點擊「完成斬擊」，跳至**勝利排名畫面**

**勝利排名畫面**顯示各隊對鬼王造成的傷害貢獻（`damageDealt`），由高至低排列：

| 名次 | 標示 |
|:----:|:-----|
| 第 1 名 | 金色高亮，稱號「最強鬼殺隊員」|
| 第 2～3 名 | 銀/銅標示 |
| 第 4～6 名 | 名次數字 |

### 5.2 鬼王勝利

所有隊伍 HP 同時歸零（含單次攻擊全滅），觸發鬼王勝利終局畫面。

## 六、系統功能

### 6.1 自動儲存

遊戲狀態自動儲存至 `localStorage`，重新開啟頁面可繼續上次進度。

### 6.2 上一步（還原）

可還原上一個操作動作（支援任意步驟復原）。

### 6.3 存檔／讀檔

產生 JSON 格式存檔供複製；貼上 JSON 字串可還原進度。存檔結構如下：

```json
{
  "ghostKingHP": 100,
  "bossPosition": 1,
  "currentTurnType": "player",
  "currentTeamIndex": 0,
  "globalRound": 1,
  "bossStatus": {
    "nextAttackMultiplier": 1,
    "isBossTurnSkip": false
  },
  "teams": [
    {
      "id": 1,
      "position": 1,
      "hp": 10,
      "damageDealt": 0,
      "status": {
        "shieldActive": false,
        "doubleDamage": false
      }
    }
    // ... 後續 5 隊數據
  ]
}
```

> 事件日誌**不納入**存檔。讀檔後顯示系統訊息「已從存檔還原，第 X 輪」。

## 七、UI 介面佈局（16:9）

整體畫面鎖定 16:9 比例，適合投影機顯示，採暗黑風格（深色背景、高對比文字）。

### 7.1 開場畫面

全螢幕顯示：遊戲大標題、副標題、六隊隊名列表（含代表色）、「開始遊戲」按鈕。

### 7.2 主遊戲畫面（三欄式）

| 區塊 | 寬度 | 內容 |
|:----:|:----:|:-----|
| 左側 | 30% | 鬼王狀態卡（HP 條、狀態標籤）；6 隊狀態列表（HP、Buff 圖標、傷害貢獻）|
| 中間 | 40% | 16 格環狀棋盤；控制中心（擲骰按鈕、任務成功/失敗、抽卡、換下一位）|
| 右側 | 30% | 當前任務公告板（超大字）；事件日誌（依事件類型高亮）|

頂部列：當前大輪數、遊戲標題、存檔/讀檔按鈕。

### 7.3 最終一擊儀式畫面

全螢幕黑底，中央血紅大字「**鬼 殺 隊 ―― 斬 ！**」，底部「完成斬擊」按鈕。

### 7.4 勝利終局畫面

全螢幕顯示傷害貢獻排行榜，第一名金色高亮。
